# [APIO2020] 交换城市

​ 给定 nnn 个点和 mmm 条边的无向图，每条边有边权。有 QQQ 次询问，每次询问 u,vu,vu,v 两点是否可以互达，要求不能相遇，若能互达最小化路径上边权的最大值，否则输出 -1，强制在线。

​ 性质：共有 2n−12n-12n−1 个节点。其中叶子节点都是原树上的节点，其余节点表示某条边的边权。原图中两个点之间的路径上最大边权的最小值为 KruskalKruskalKruskal 重构树上两个节点的 LCALCALCA 的点权。（若求两个节点的最小边权的最大值跑最大生成树即可）

发现这道题我们额外做的就是在做 KruskalKruskalKruskal 重构树的时候去维护这些连通块的信息，我们主要考虑去处理链的情况。假如我们要处理边 (u,v,w)(u,v,w)(u,v,w) ，再设 fau=p,fav=qfa_u=p,fa_v=qfau​=p,fav​=q 。

当 u,vu,vu,v 不连通时，我们发现，若其中有一个为非链，那么合并完也必然非链，所以我们直接对为链的那个连通块（也可能没有）进行连边操作，合并两个连通块并将标记设为非链形态。但是也有可能两个都是链，那我们还需要分类讨论一下。若点 u,vu,vu,v 都是两条链的某个端点，那它合并完仍然是一条链，所以我们直接修改新链的 p1,p2p1,p2p1,p2 两端点即可。否则，它合并后一定为非链，直接将两个连通块全进行加边操作即可。

此时我们就完美用 KruskalKruskalKruskal 重构树解决了询问，最后我们只需要求两点 LCALCALCA 即可，其点权就是答案了。注意最后建出的是一颗森林，则无解当且仅当两个点不在同一棵树上，这个 dfsdfsdfs 的时候直接处理一下即可。总时间复杂度为 O(nlog⁡n+Qlog⁡n)O(n\log n+Q\log n)O(nlogn+Qlogn) 。

​ 构造一个含有 kkk 个上升子序列的排列（从 000 开始，包括空序列）。要求序列长度 ≤90\le 90≤90 。

先说一个很无脑的想法，直接构造一个长度为 kkk 的递减序列。可以获得 101010 分的成绩。

我们考虑 111 到 xxx 的一个递增序列，发现其任意一个子序列都是递增的，所以有 2x2^x2x 个上升子序列。由此我们可以想到将 kkk 二进制拆分，先假设其最大项为 2x2^x2x ，那么我们可以先来一个 000 到 x−1x-1x−1 。每出现一个 2x2^x2x ，我们就在当前序列的第 xxx 项后面添加 111 个严格大于当前序列中所有数的数。举个例子，现在 k=24+22+20k=2^4+2^2+2^0k=24+22+20 ，初始序列为 0,1,2,30,1,2,30,1,2,3 ，我们要出现一个 222^222 ，那我们在 111 的后面加一个 444 ，变成 0,1,4,2,30,1,4,2,30,1,4,2,3 ，这样 444 就可以产生 222^222 个上升子序列，对于 202^020 ，我们在最前面放一个 555 即可。最终得到 5,0,1,4,2,35,0,1,4,2,35,0,1,4,2,3 。按照这样最后序列的长度为 log⁡2k+popcount(k)−1\log_2 k+popcount(k)-1log2​k+popcount(k)−1 。但发现 1018<26010^{18}<2^{60}1018<260 ，那么对于 259−12^{59}-1259−1 ，它就需要 58+58−1=11558+58-1=11558+58−1=115 个数，还是超出了限制，我们继续考虑优化。

发现最开始的那 log⁡2k\log _2klog2​k 个数的骨架是很难减掉了。发现 90=1.5×6090=1.5\times 6090=1.5×60 ，我们就有一个非常妙的想法：能不能把两个 111 尝试用一个 111 表示出来。发现这个想法是可行的！我们以序列 0,4,1,6,2,5,30,4,1,6,2,5,30,4,1,6,2,5,3 为例，初始序列是 0,1,2,30,1,2,30,1,2,3 ，产生 242^424 个上升子序列。然后我们加了一个 555 ，产生了 232^323 个子序列，然后又加了一个 666 ，产生了 222^222 个子序列，然后我们又加了一个 444 ，它可以和 000 以及后面的 6,56,56,5 共产生 20+20×2=20+212^0+2^0\times 2=2^0+2^120+20×2=20+21 个子序列。这样这个 444 就起到了用一个表示两个的作用。一般地，在我们得到初始序列后，我们按二进制位从大到小先找到两个 111 作为这个操作的基础，并添加两个递减的数，记录下最大值和次大值的位置。然后我们继续往后找 111 ，每找到两个连续的 111 ，我们就可以按照刚刚例子的操作，先把那两个最大值和次大值加 111 ，然后在对应的位置添加一个原来作为次大值的值，这压根就可以产生 333 倍的贡献。而如果不是两个连续的 111 ，我们就按照上面的方案，直接插入一个比当前最大值加 111 的数，并修改最大值和次大值位置即可。这样序列长度就可以控制在 1.5log⁡k1.5\log k1.5logk 了，可以得到满分。

# [APIO2018] 新家

​ 在一条数轴上会出现 nnn 个点，每个点有颜色、位置、出现时间和消失时间，共有 kkk 种颜色。接下来有 QQQ 个询问，每次询问为：在时间 ttt 对于位置 pospospos ，每种颜色的点到 pospospos 的最小距离的最大值。若在时间 ttt 并不存在所有 kkk 种颜色的点，输出 -1。

​ 1≤n,Q≤3×1051\le n,Q\le 3\times 10^51≤n,Q≤3×105 ，1≤k≤n1\le k\le n1≤k≤n 。

位置和时间的值域范围为 [1,108][1,10^8][1,108] 。

我们先考虑怎么处理时间的限制，这很容易想到将所有点的出现时间、消失时间以及询问一起离线下来排个序。然后我们考虑这种区间数颜色的操作，经典的操作是对于每个位置记录与其颜色种类相同的上一个位置，而这里由于每个位置上可能出现多个点，所以我们每个位置（也就是叶子节点）开一个 multisetmultisetmultiset 维护该位置上所有颜色的点对应的前驱，区间直接维护其前驱最小值。同时我们对于每种颜色的点还要开一个 multisetmultisetmultiset 记录其出现的位置，还要先插入两个哨兵商店，这样可以很方便的查找其前驱后继。具体地，若新加一个点位置为 pospospos ，我们找到该点的前驱 aaa 和后继 bbb ，我们将 bbb 的前驱修改为它自己，也就是将位置 bbb 的 multisetmultisetmultiset 删去 aaa 而插入 pospospos ，再将 pospospos 的前驱修改为 aaa ，再看其 sizesizesize 是否为 222 ，若是 222 则说明该种颜色的点原先没有而现在有了，所以该时刻的颜色种类数要加一，同时对应颜色的 multisetmultisetmultiset 还要插入 pospospos 。删除的处理与插入类似。对于询问，我们可以通过二分长度 lenlenlen ，若 [x−len,x+len][x-len,x+len][x−len,x+len] 包含了所有的 kkk 种颜色，那么 lenlenlen 合法。怎么判断是否包含所有颜色呢，我们利用记录的前驱最小值，若 r+1r+1r+1 往后的位置前驱最小值小于 lll ，说明一定有某种颜色没有在 [l,r][l,r][l,r] 内出现。这样单次时间复杂度为 log⁡2n\log ^2nlog2n 。我们可以直接线段树上二分位置，这样时间复杂度可以做到 O(nlog⁡n+Qlog⁡n)O(n\log n+Q\log n)O(nlogn+Qlogn) 。

# [APIO2018] 铁人两项

​ 给定一张 nnn 个点 mmm 条边的无向图，问有多少三元组 (a,b,c)(a,b,c)(a,b,c) 满足存在一条简单路径，从 aaa 出发经过 bbb 到达 ccc 。

​ n≤105n\le 10^5n≤105 ，m≤2×105m\le 2\times 10^5m≤2×105 。

我们发现当 aaa 和 ccc 确认时 bbb 的数量就是 aaa 到 ccc 的所有简单路径的并的大小减二（减去 aaa 和 ccc 两个点）。此时可以用到点双连通分量的一个性质：对于同一个点双中的两个点其简单路径的并集恰好为整个点双。所以 aaa 到 ccc 的简单路径的并还等于 aaa 到 ccc 经过的所有点双的并集。我们直接建出原图的圆方树，这样两点间路径就和其在圆方树上的路径经过的圆点和方点有关。这里我们运用一个小技巧，对圆方树上的点赋上点权。我们在这里将方点点权设为点双的大小，圆点点权设为 -1。手模一下发现此时两点间简单路径的并的大小减二就等于圆方树上两点路径上经过的点的权值和。此时问题就转化为统计圆方树上两圆点间路径的权值和。这是经典问题，我们可以考虑每个点对答案的贡献，直接 dfs 即可。时间复杂度为 O(n+m)O(n+m)O(n+m) 。
