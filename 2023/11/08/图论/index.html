# CF1515F Phoenix and Earthquake

给定一张 nnn 个点 mmm 条边的无向连通图和正整数 xxx ，每个点有非负点权 wiw_iwi​ 。如果一条边 (u,v)(u,v)(u,v) 满足 wu+wv>xw_u+w_v>xwu​+wv​>x 则可以将这两个点缩起来，新点的点权为 wu+wv−xw_u+w_v-xwu​+wv​−x 。判断这张图能否缩成一个点，如果可以还需输出每次缩的是哪条边。

2≤n,m≤3×1052\le n,m\le 3\times10^52≤n,m≤3×105 。

先摆出结论：如果 ∑i=1nwi<(n−1)×x\sum_{i=1}^{n}w_i\;<(n-1)\times x∑i=1n​wi​<(n−1)×x 则无解，考虑归纳证明。在 nnn 个点的树中，对于叶子节点 uuu ，若 wu≥xw_u\ge xwu​≥x ，那我们直接将 uuu 和 faufa_ufau​ 缩起来，变成一个 n−1n-1n−1 个点的树，此时仍满足条件。否则若 wu<xw_u<xwu​<x ，我们直接删掉点 uuu 和连向 faufa_ufau​ 的边，大数减小数大于小数减大数，所以剩下的 n−1n-1n−1 个点的树仍满足条件，我们只需要最后缩点 uuu 即可。此时做法已经很显然了，搞出来这个图的一颗生成树， dfsdfsdfs 一遍，wu≥xw_u\ge xwu​≥x 的点加入答案队列中，然后将权值修改，否则将其加入答案栈中，最后先输出队列后输出栈内的边即可。时间复杂度 O(n)O(n)O(n) 。

# CF1517D Explorer Space

给定一个四联通的 n×mn\times mn×m 的带边权网格图，对于每个节点求恰好 kkk 步回到自身的最短路径，若无法到达输出 −1-1−1 。

1≤n,m≤5001\le n,m\le 5001≤n,m≤500 ，k≤20k\le 20k≤20 。

发现只有当 kkk 是奇数的时候才无解，所以 kkk 为偶数的时候就转化成走 k2\frac{k}{2}2k​ 步，且走出去和走回来的路径可以相同，所以最终转化为走 k2\frac{k}{2}2k​ 步的最小权值。很明显可以直接 dpdpdp ，设 fi,j,kf_{i,j,k}fi,j,k​ 表示当前在点 (i,j)(i,j)(i,j) 走了 kkk 步的最小权值。转移对于每个节点 (i,j)(i,j)(i,j) 枚举其四联通的节点转移过来即可。时间复杂度为 O(nmk)O(nmk)O(nmk) 。

# 洛谷 P4320 道路相遇

给定 nnn 个点 mmm 条边的无向连通图。QQQ 次询问两点之间的必经点数量（包括出发点和终点）。

n≤5×105n\le 5\times 10^5n≤5×105 ，Q≤5×105Q\le 5\times 10^5Q≤5×105 ，m≤min⁡(n(n−1)2,106)m\le \min(\frac{n(n-1)}{2},10^6)m≤min(2n(n−1)​,106) 。

容易发现两点间的必经点个数就是路径上割点的个数。所以我们直接建圆方树，发现就是求两点间路径上圆点的数量，由于圆方树上路径是圆点和方点间隔分布，所以答案就是两点间路径长度除以二加一，直接对圆方树做树剖即可。时间复杂度为 O(nlog⁡n+m+Qlog⁡n)O(n\log n+m+Q\log n)O(nlogn+m+Qlogn) 。

# CF1051F The Shortest Statement

给定 nnn 个点 mmm 条边的带边权无向图，满足 m−n≤20m-n\le 20m−n≤20 ，QQQ 次询问两点间的最短路径。

n,m≤105n,m\le 10^5n,m≤105 ，Q≤105Q\le 10^5Q≤105 。

由于 m−n≤20m-n\le 20m−n≤20 的性质，我们容易发现给出的无向图可以拆成一棵生成树和单独的 202020 多条边，所以两点间最短路径可以分为两种，一是只在最小生成树上的，二是包含非最小生成树上的边。对于一，我们对最小生成树做树剖将两点间路径拆成与 lcalcalca 有关的形式即可，对于二，我们记录下非最小生成树上的边，对于这些边的左右端点，我们以它们为起点跑单源最短路径即可。时间复杂度为 (m−n+1)log⁡n+nlog⁡n+Qlog⁡n(m-n+1)\log n+n\log n+Q\log n(m−n+1)logn+nlogn+Qlogn 。

# CF732F Tourist Reform

给定一个 nnn 个点 mmm 条边的无向连通图，要求将所有边定向，设 fif_ifi​ 表示点 iii 可以到达的点的数量，最大化 min⁡i=1nfi\min \limits_{i=1}^{n}{f_i}i=1minn​fi​ ，输出最大值和每条边的方向。

n,m≤4×105n,m\le 4\times 10^5n,m≤4×105 。

我们容易发现，对于一个边双连通分量，一定存在一种定向方案使得其中任意两个节点能够互达，即它们的 fff 值为这个边双的大小。所以我们对原图进行边双缩点，这样会得到一棵树，每条边就对应原图的一个桥。现在我们单独来看树的情况，假设它有 nnn 个点，因为它有 n−1n-1n−1 条边，那么定向后它的出度之和最大为 n−1n-1n−1 ，所以一定存在某个点它的出度是 000 ，也就是说这个点的 fff 值为 000 。回到原题上来，也就是一定有某个边双，其内部的点的 fff 值就是这个边双的大小。那么最大值就是所有边双的大小的最大值了。对于输出方案，我们从大小最大的那个边双里的一个点直接 dfsdfsdfs 即可。时间复杂度为 O(n+m)O(n+m)O(n+m) 。

# 洛谷 P4366 [Code+#4] 最短路

考虑优化第一部分的完全图建图。假如我们要从 111 走到 777 ，如果直接走，代价是 6C6C6C 。而如果我们按位去走，先从 111 走到 333 ，代价为 2C2C2C ，再从 333 走到 777 ，代价为 4C4C4C ，总的代价仍然是 6C6C6C 。这其实启发我们完全可以只保留只变化一个二进制位的边，这样总代价是不会变化的，那么就只需要保留从 xxx 连向 x⨁2kx\bigoplus2^kx⨁2k 的边，但要满足 x⨁2k≤nx\bigoplus2^k\le nx⨁2k≤n 。而如果点的编号出现了 000 该怎么办呢，事实上 000 不会影响答案，因为 (u⨁0)+(0⨁v)=u+v≥u⨁v(u\bigoplus 0)+(0\bigoplus v)=u+v\ge u\bigoplus v(u⨁0)+(0⨁v)=u+v≥u⨁v 。这样整张图的边数就降到了 O(nlog⁡n+m)O(n\log n+m)O(nlogn+m) 级别，然后再用 dij 跑最短路即可。

# gym104197D/QOJ5520 Distance Parities

如果有解，那么直接将距离为奇数的两个点之间连一条边也一定是合法的。考虑在某组合法解中两个点 u,vu,vu,v 之间的距离为偶数 2k2k2k ，那么在这两个点的最短路上一定存在一个点 ppp ，使得 uuu 到 ppp 的距离为 111 ，ppp 到 vvv 的距离为 2k−12k-12k−1 ，也是奇数。而按照我们的做法，我们一定会连一条从 uuu 到 ppp 和从 ppp 到 vvv 的边，此时新图中 uuu 到 vvv 的距离为 222 ，仍然是偶数。因此我们只需要建完图后跑一遍 floyd 判断两点间距离是否满足给出的奇偶性，以及是否是连通图即可。

# gym104023F Mooncake Delivery

不妨考虑答案的形态，由于我们每走到一个点就可以回收其它颜色的所有顶点的点权，那么我们要将路径按照颜色去分段，那么初始需要的点权就是每一段的点权和加上下一段的起始点的点权的最大值。于是我们先按照每种颜色将图分成几个同色连通块，每块内部用 floyd 求出两两之间最短路。然后就可以考虑枚举每一段的起点 uuu ，终点 vvv 和下一段的起始点 www ，连边 (u,w,disu,v+valw)(u,w,dis_{u,v}+val_w)(u,w,disu,v​+valw​) ，建出一张新的有向图。然后在这张新图上跑一个边权取 max⁡\maxmax 的 floyd 即可。

# QOJ6306 Chase Game

可以发现当第二个人瞬移过来后，第一个人就一定是沿着最短路直接走到 nnn 了，此时这一部分的代价可以直接算出来。而至于第二个人还在点 kkk 的这一部分，可以给每个点赋上 D−disk,uD-dis_{k,u}D−disk,u​ 的点权然后跑 dij 求出到达某个点的最小代价。那么答案就分三种情况，一是只有第一部分，此时就是 dij 跑出来的到 nnn 的最短路；二是走了一步就直接进入第二部分，可以通过枚举 111 的出点求出；三是两个部分都有，枚举中转点和中转点的出点，两个部分代价相加即可。注意还需要两次 bfs 预处理出每个点到点 nnn 和点 kkk 的最短路。

# 洛谷 P3623 [APIO2008] 免费道路

先对黑色边做一遍生成树，再把白色边加进去，那么此时加进去的白边是一定要加的。那么我们先将一定要加的那些白边加上，然后再补齐 kkk 条白边，最后把剩下的黑边加进去就行，本质上就是每次加一条白边产生环后删去一条环上的黑边。无解情况就是必须加的白边数量 ≥k\ge k≥k 或者加不够 kkk 条白边或者图不连通。

# 洛谷 P3639 [APIO2013] 道路费用

容易想到 2k2^k2k 去枚举这 kkk 条边的一个使用情况，然后再加入原图中的边来保持连通性，但复杂度显然有点高。我们不妨先只连上这 kkk 条边，然后尝试将原图中的 mmm 条边加入，直到图连通，此时后加入的那些原图上的边就是必须要加的，因为其它方案加入的新边的数量少了，那么这些后加入的边肯定也得选上，然后还可能再加另一些边。由于保证边权互不相同，因此后加入的那些边是固定的，所以可以将这些边先缩起来，缩完后点的数量就变成了 k+1k+1k+1 。同时由于新边数量的减少，所以可能还需要加入一些原图上的边，我们再把备用的边全部找出来，显然备用的边只会有 kkk 条。此时就可以套用原来的做法，2k2^k2k 去枚举选择情况，先加入新边，再加入备用的边使得整张图连通，然后对于计算新加入的边的代价，先考虑这些边的边权，事实上那些没有加入的备选边对新加入的边的边权进行了限制，它要求最小生成树上那两点之间的每条边的边权都必须小于它的边权，否则将最大的删去替换成它一定更优。由于点数很小，所以我们可以暴力跳路径求出每条边的最大边权。至于每条边的经过次数，以 111 所在的点为根 dfs 求出每个点子树内的人数即可。总时间复杂度为 O(mlog⁡m+2kk2)O(m\log m+2^kk^2)O(mlogm+2kk2) 。

# CF1628E Groceries in Meteor Town

前置题目：CF1062E Company 结论：一些节点的 LCA 一定是其中 dfs 序最小和 dfs 序最大的两个点的 LCA 。那么只需要对于每个询问区间，枚举是删掉 dfs 序最小的点还是 dfs 序最大的点，删掉后可以分成两个子区间，求出两个子区间的 LCA ，再求一下两个子区间的 LCA 的 LCA 就是删掉这个点后这段区间的 LCA ，然后两种情况比一下深度就行。

我们回到这道题上，它要求节点 xxx 到任意一个白色节点的简单路径上经过的边的最大权值，容易想到建 Kruskal 重构树，那么问题就转变成询问 xxx 与所有白色节点的 LCA 。根据上面的结论，可以转化成求白色节点与 xxx 中 dfs 序的最小值和最大值，可以用线段树进行维护，每个节点维护一个区间内白色节点的 dfs 序最小值和最大值，所有节点的 dfs 序最小值和最大值，以及一个更新的懒标记即可。

# gym103427H/QOJ6019 Line Graph Matching

大力手玩可以发现当边数是偶数时，所有边都是可以匹配的。而至于奇数的情况，我们显然是舍弃掉边权最小的那一条边。但是这条边也有一些限制，考虑如果这条边是割边，对于删掉它后形成的两个连通分量，是需要满足边数都是偶数的，如果不是割边就没有别的要求了。因为如果都是奇数我们就还得删别的边，那么显然是不优的。然后就在符合条件的边集中选一条最小的删掉即可。
