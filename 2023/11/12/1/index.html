# gym104768K / QOJ7687 Randias Permutation Task

容易发现答案最多为 min⁡(n!,2m)\min(n!,2^m)min(n!,2m) ，由于 n×m≤180n\times m\le 180n×m≤180 ，那么答案 ≤9!=362880\le 9!=362880≤9!=362880 。因此复杂度为 O(ans)O(ans)O(ans) 级别的爆搜都是对的。维护一下当前的答案集合，枚举下一个排列是选还是不选即可。

# CF1468M Similar Sets

看到 ∑ki≤2×105\sum k_i\le 2\times 10^5∑ki​≤2×105 ，考虑根号分治。下设 m=∑kim=\sum k_im=∑ki​ ，我们考虑那些大小超过 m\sqrt{m}m​ 的序列，显然这样的序列不会超过 m\sqrt{m}m​ 个，我们每次可以暴力遍历除了它以外的所有序列进行判断，这部分复杂度是 O(mm)O(m\sqrt{m})O(mm​) 的。现在我们再只考虑那些大小 <m<\sqrt{m}<m​ 的序列，我们可以暴力枚举这些数列的所有元素对 (x,y)(x,y)(x,y) ，并把它们全部储存起来然后判断是否有两个相同数对即可。这里有一个常用结论：** 将 nnn 分成若干个不超过 xxx 的数字，它们的平方和的最大值约等于 nxnxnx ** 。则元素对的数量级别是 O(mm)O(m\sqrt{m})O(mm​) 的，那么这一部分复杂度也是 O(mm)O(m\sqrt{m})O(mm​) 的。因此总复杂度也是 O(mm)O(m\sqrt{m})O(mm​)

# ABC259Ex Yet Another Path Counting

首先有一个显然的暴力，对于每种颜色找到其所有格子，然后枚举起点和终点用组合数 O(1)O(1)O(1) 计算答案。然后还能想到一种 dp 做法，就是对于每种颜色，设 fi,jf_{i,j}fi,j​ 表示所有可能的起点走到 (i,j)(i,j)(i,j) 的方案数，转移是平凡的。我们考虑将它们综合起来，考虑题目中一个隐藏的限制，∑cnti=n2\sum cnt_i=n^2∑cnti​=n2 ，我们不妨将两种方法结合一下。我们设立一个阈值 BBB ，当这个颜色对应的格子数量 ≥B\ge B≥B 时，这样的格子只会有 n2B\frac{n^2}{B}Bn2​ 个，我们用第二种方法，时间复杂度为 O(n4B)O(\frac{n^4}{B})O(Bn4​) ；当格子数量 <B<B<B 时，我们用第一种方法，根据上一题的结论，复杂度是 O(nB)O(n^B)O(nB) 的。平衡一下复杂度，取 B=nB=nB=n 即可，总时间复杂度为 O(n3)O(n^3)O(n3) 。

# QOJ6405 Barkley

不妨先考虑 k=1k=1k=1 的情况，设 g(l,r)g(l,r)g(l,r) 表示区间 [l,r][l,r][l,r] 的 gcd⁡\gcdgcd ，那么就相当于求 max⁡x=lrgcd⁡(g(l,x−1),g(x+1,r))\max\limits_{x=l}^{r}\gcd(g(l,x-1),g(x+1,r))x=lmaxr​gcd(g(l,x−1),g(x+1,r)) 。考虑左端点固定后 gcd⁡(l,x)\gcd(l,x)gcd(l,x) 会形成一些连续段，那么显然删除突变点是最优的，即每个不同取值第一次出现的位置。并且由于段的个数只会有 log⁡V\log VlogV 级别，因此我们直接倍增找出每个连续段的开头位置，然后求出删掉它的答案并不断取 max⁡\maxmax 即可。那么对于 kkk 更大的情况，容易发现删掉一个位置后右边的区间可以变成一个 k−1k-1k−1 的子问题，因此写成递归的形式即可。总时间复杂度为 O(nlog⁡2V+∑log⁡k+1V)O(n\log^2V+\sum \log^{k+1}V)O(nlog2V+∑logk+1V) 。

# gym104768I / QOJ7685 Barkley II

考虑去枚举 mex=xmex=xmex=x ，那么我们其实只需要找出所有不出现 xxx 的极长区间即可，只需要简单地记录 xxx 上一次出现得位置就能找出这些区间，然后对于这些区间都做一遍区间数颜色。显然总共的区间个数不会超过 2n2n2n ，离线下来扫描线加树状数组即可，最后对于每个枚举到的 mexmexmex ，用最多的区间颜色数减去它去更新一下答案。至于正确性，因为如果真实的 mexmexmex 更小，那么一定有一个更长的区间去对应于这个 mexmexmex ，因此答案不会更劣。时间复杂度为线性对数。

# gym104768C / QOJ7679 Master of Both IV

设 x=lcm(S)x=lcm(S)x=lcm(S) ，y=xor(S)y=xor(S)y=xor(S) ，那么显然有 x≥max⁡(S),y<2×max⁡(S)x\ge \max(S),y<2\times \max(S)x≥max(S),y<2×max(S) ，所以可能的情况只有两种，y=0y=0y=0 或者 y=max⁡(S)y=\max(S)y=max(S) 。考虑 y=0y=0y=0 的情况，事实上就是问有多少个非空子集异或和为 000 ，这是一个经典结论，假设所有数字组成的线性基大小为 xxx ，那么数量就是 2n−x−12^{n-x}-12n−x−1 ，因为考虑不在线性基里的 n−xn-xn−x 个元素，对于它们的任何一个非空子集的异或和，我们一定能从线性基中用唯一的方案异或出这个数，使得这两部分的总异或和为 000 。而对于第二种情况，我们可以直接枚举最大值 xxx 。方案数分成两部分，对于 xxx 这个数，就必须选奇数次，设 cntxcnt_xcntx​ 表示 xxx 的出现次数，那么方案数就是 (cntx1)+(cntx3)+⋯=2cntx−1\binom{cnt_x}{1}+\binom{cnt_x}{3}+\cdots=2^{cnt_x-1}(1cntx​​)+(3cntx​​)+⋯=2cntx​−1 ；而剩下的数就必须是 xxx 的约数（不包含自己），我们需要从这些约数中选一些子集使得其异或和为 000 ，这又变成了第一部分的那个问题，仍然用线性基解决即可，然后用乘法原理将两部分方案数合并即可。总时间复杂度为 O(nlog⁡2V)O(n\log^2V)O(nlog2V) 。

# CF1468L Prime Divisors Selection

先特判掉 k=1k=1k=1 的情况，此时显然无解。我们将所有数分成两类，A 类是只有一个质因子的，B 类是有多个质因子的。如果一个质因子在 A 类中的出现次数 ≤1\le 1≤1 ，那么包含这个质因子的所有数都不能被选择，我们将这些质因子都扔掉。此时留下来的只有那些在 A 类中出现次数 ≥2\ge 2≥2 的质因子组成的数，那么当 kkk 大于剩下的数的数量也是无解的了。我们考虑剩下的情况，下设 cntcntcnt 表示出现次数 ≥2\ge 2≥2 的质因子数量。如果 k≥2×cntk\ge 2\times cntk≥2×cnt ，我们就可以直接对每个质因子都选两个 A 类，然后剩下的随便选一些 B 类即可；否则如果 kkk 是偶数，只需要随便 k2\frac{k}{2}2k​ 个质因子，然后每个质因子随便选两个 A 类即可。剩下的就是 kkk 是奇数且 k<2×cntk<2\times cntk<2×cnt 的情况，容易想到，如果某个质因子在 A 类的出现次数 ≥3\ge 3≥3 ，那我们就选三个包含它的 A 类数，然后就又变成 kkk 是偶数的情况了。那么剩下的情况就是每个质因子在 A 类的出现次数都等于 222 ，此时我们考虑利用 B 类数，我们找到质因子个数最小的一个 B 类数 xxx ，假设它有 p1,p2⋯pmp_1,p_2\cdots p_mp1​,p2​⋯pm​ 共 mmm 个质因子，对于这 mmm 个质因子当中的每一个，我们都选出两个包含它的 A 类数，然后还要再选上 xxx ，此时就共选了 2m+12m+12m+1 个数，因此如果 k<2m+1k<2m+1k<2m+1 就无解，否则由于 kkk 是奇数，因此我们还需要选偶数个，按照偶数的做法选即可。

但是还有一个问题，我们可不可以不用大数分解算法去分解质因数呢，其实是可以的。首先只有 <109<10^9<109 的质数才是有用的，因为其它质数的出现次数最多为 111 。我们再线性筛求出 10610^6106 以内的质数，用它们去找出每个 A 类质数。而对于剩下的数，要不是一个 ≤109\le 10^9≤109 的质数的平方，要不是一个质数。因此我们直接对出现过的 ≤109\le 10^9≤109 的数判断它的平方是否出现在序列中就行，剩下的数就一定没用了，这样我们就避免了使用大数分解算法。实现起来比较麻烦。

# gym104768J / QOJ7686 The Phantom Menace

不妨去枚举循环同构的偏移量 xxx ，容易发现就是要求第一个集合中的字符串的长度为 xxx 的后缀与第二个集合中的字符串的长度为 xxx 的前缀匹配，且第二个集合中的字符串的长度为 n−xn-xn−x 的后缀与第一个集合中的字符串的长度为 n−xn-xn−x 的前缀匹配。哈希去重后我们对每个前后缀编号，然后将前后缀看成点，让每个串的前缀向后缀连边，不难发现只需要求解有向图欧拉回路即可。时间复杂度 O(nm)O(nm)O(nm) 。

# QOJ2831 Game Theory

不妨先找到初始状态下的 kkk ，然后考虑 aka_kak​ 。如果 ak=1a_k=1ak​=1 ，则操作后 111 的个数减少，kkk 会向左移动；如果 ak=0a_k=0ak​=0 ，则操作后 111 的个数增多，kkk 会向右移动。那么移动路径一定是先左右来回运动，每次遇到一个不同的就反向，最后一路走到头全部变成 000 。观察到 kkk 左边 000 的个数是等于右边 111 的个数的，并且可以发现，其实答案就是 kkk 右侧的 111 的下标之和减去 kkk 左侧的 000 的下标之和的差乘 222 再加上 kkk ，化简一下其实就是整体的 111 的下标之和乘 222 减去 k2k^2k2 ，因此线段树维护区间中 111 的下标之和、区间中 111 的数量以及一个翻转的懒标记即可。时间复杂度 O((n+q)log⁡n)O((n+q)\log n)O((n+q)logn) 。

# CF1468B Bakery

先将询问离线下来。显然答案会随着 kkk 的增加而减少，考虑当 kkk 减少时去维护这几个连续段，同一个连续段内的可以把生产的面包全部卖完，如果因为 kkk 的减少导致面包没有卖完，我们就让这段不断向后合并即可。那么每个连续段显然需要记录它总共生产的面包数量和总共的天数，这样判断一下天数乘 kkk 与总共生产的面包数量的大小关系就知道是否需要合并了，这些连续段可以用一个 set 维护，按照总共生产的面包数量除以天数从大到小排序就行。然后连续段的合并也可以用并查集简单维护，每次询问的答案就是区间长度的 max⁡\maxmax 了，注意还要特殊处理右端点为 nnn 的情况。
